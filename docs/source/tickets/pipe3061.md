# `@pytest.mark.mpi` usage examples

## Overview

A `mpi` pytest marker was added to tag tests that are not required but recommended
for an MPI-enabled CASA session (`mpicasa`). There is no auto-skip logic — use explicit `-m` expressions
to control which tests run in a given session.

In this environment, invoking `pytest` directly under `mpicasa` is not reliable.
Use `mpicasa -n N casa -c "import pytest; pytest.main([...])"` for MPI sessions.

Marker registration (`pyproject.toml`):

```toml
"mpi: test recommended for an MPI-enabled CASA session; use -m mpi or -m 'not mpi' to control execution"
```

---

## Serial session: exclude mpi tests

```bash
# Run all fast regression tests, skipping mpi-marked ones
pytest -m "not mpi" tests/regression/fast/

# All regression tests, no mpi
pytest -m "not mpi" tests/regression/
```

---

## Serial session: run mpi tests anyway (debugging)

No flag needed — just use `-m mpi` directly.  Be aware the test will likely
fail or produce incorrect results without a real MPI environment.

```bash
# Force-run a single mpi test in serial (expect failures)
pytest -m mpi tests/regression/fast/alma_if_fast_test.py::test_2022_1_00207_S__uid___A001_X2d20_X373d__PPR__regression

# All mpi tests, serial (expect failures)
pytest -m mpi tests/regression/fast/
```

---

## mpicasa session: run only mpi tests

```bash
mpicasa -n 4 casa --nogui --log2term \
  -c "import pytest; pytest.main(['-m', 'mpi', 'tests/regression/fast/'])"

# Scope to a specific file
mpicasa -n 4 casa --nogui --log2term \
  -c "import pytest; pytest.main(['-m', 'mpi', 'tests/regression/fast/alma_if_fast_test.py'])"
```

---

## mpicasa session: run only non-mpi tests

```bash
mpicasa -n 4 casa --nogui --log2term \
  -c "import pytest; pytest.main(['-m', 'not mpi', 'tests/regression/fast/'])"
```

---

## Collect tests only (no execution)

The `--collect-tests` flag writes all collected node IDs to `collected_tests.txt`.
All tests including `mpi`-marked ones are always collected regardless of `-m`.

```bash
# Collect all fast tests
pytest --collect-tests tests/regression/fast/

# Collect everything, then inspect for mpi tests
pytest --collect-tests tests/
grep mpi collected_tests.txt
```

## Combining with other markers

```bash
# All ALMA fast tests, no mpi, serial session
pytest -m "alma and fast and not mpi" tests/

# Only ALMA + mpi tests under mpicasa
mpicasa -n 4 casa --nogui --log2term \
  -c "import pytest; pytest.main(['-m', 'alma and mpi', 'tests/'])"

# Slow mpi tests under mpicasa
mpicasa -n 4 casa --nogui --log2term \
  -c "import pytest; pytest.main(['--longtests', '-m', 'mpi', 'tests/regression/'])"
```

## Summary table

| Session type | Want to run    | Command                                           |
|--------------|----------------|---------------------------------------------------|
| serial       | non-mpi tests  | `pytest -m "not mpi" tests/`                      |
| serial       | mpi tests only | `pytest -m mpi tests/`  *(expect failures)*       |
| mpicasa      | mpi tests only | `mpicasa -n N casa -c "import pytest; pytest.main(['-m', 'mpi', 'tests/'])"` |
| mpicasa      | non-mpi tests  | `mpicasa -n N casa -c "import pytest; pytest.main(['-m', 'not mpi', 'tests/'])"` |

## Incompatibility: pytest-xdist (`-n`) inside mpicasa

**Do not combine `mpicasa` with `pytest -n N` (xdist).** They are incompatible:

- xdist spawns N independent subprocesses via Python's `multiprocessing`.
  These are **not** MPI ranks — inside each worker `MPIEnvironment.is_mpi_enabled`
  is `False`, so MPI code paths silently fall back to serial behaviour.

- MPI collective operations (barriers, broadcasts, reduces) require **all ranks
  to participate together**.  With xdist running independent tests concurrently,
  ranks will be in different pipeline tasks at different times and will
  **deadlock** waiting for each other.

| Tool | Parallelism unit | Designed for |
|------|-----------------|--------------|
| `mpicasa -n N` | MPI ranks within one pipeline run | Parallelize work *inside* a single test |
| `pytest -n N` (xdist) | Independent test worker processes | Run *multiple tests* concurrently |

### Safe combinations

```bash
# MPI-parallelized pipeline, one test at a time (correct)
mpicasa -n 8 casa --nogui --log2term \
  -c "import pytest; pytest.main(['-m', 'mpi', 'tests/regression/fast/'])"

# Multiple serial tests in parallel (correct)
pytest -n 4 -m "not mpi" tests/regression/fast/

# DO NOT DO THIS — deadlock / wrong results
# mpicasa -n 8 casa -c "import pytest; pytest.main(['-n', '4', 'tests/regression/fast/'])"
```

## Simplified invocation example

If your runtime wrapper generates a very long command line, use a reduced form like
this in docs and notes.

```bash
# Serial CASA session with xdist (non-mpi tests only)
PYTHONNOUSERSITE=1 OMP_NUM_THREADS=1 OPENBLAS_NUM_THREADS=1 \
  xvfb-run -d --server-args='-nolisten inet6' \
  /path/to/casa --nogui --log2term \
  -c "import pytest; pytest.main(['-vv', '-n', '8', '--dist', 'worksteal', '-m', 'not mpi', 'tests/regression/fast/'])"
```

```bash
# MPI session (mpi tests): use mpicasa ranks, no pytest -n
mpicasa -n 8 casa --nogui --log2term \
  -c "import pytest; pytest.main(['-m', 'mpi', 'tests/regression/fast/'])"
```

This keeps the key behavior visible while avoiding a hard-to-read full launcher
string.
