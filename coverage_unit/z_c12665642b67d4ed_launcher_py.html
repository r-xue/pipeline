28class Context: 

29 """ 

30 Context holds all pipeline state, consisting of metadata describing the 

31 data set, objects describing the pipeline calibration state, the tree of 

32 Results objects summarising the results of each pipeline task, and a 

33 small number of internal pipeline variables and objects. 

34 

35 The aim of the Context class is to provide one central object to which all 

36 pipeline state is attached. Keeping all state in one object makes it easy 

37 to persist this one object, and thus all state, to disk as a Python pickle, 

38 allowing pipeline sessions to be interrupted and resumed. 

39 

40 Attributes: 

41 name: Name of the context; also forms the root of the filename used for 

42 the pickled state. 

43 

44 output_dir: The working directory where pipeline output data should be saved. 

45 products_dir: The directory where pipeline products should be exported to. 

46 report_dir: The directory where pipeline HTML reports should be saved. 

47 

48 logs: Dictionary containing the filenames to use for standard output 

49 files (CASA commands log, AQUA report, pipeline script, restore script). 

50 

51 results: The list of task result (proxy) objects containing summaries 

52 of each executed task in the pipeline run. 

53 task_counter: Index of the last task to complete. 

54 subtask_counter: Index of the last subtask to complete. 

55 

56 observing_run: The top-level ObservingRun object through which all other 

57 pipeline.domain objects can be accessed. 

58 

59 project_performance_parameters: ALMA OUS project performance information. 

60 project_structure: ALMA project structure information. 

61 project_summary: Project summary information. 

62 

63 calimlist: The ImageLibrary object holding final images of calibrators. 

64 callibrary: The CalLibrary object holding the calibration state for 

65 registered measurement sets. 

66 clean_list_info: Dictionary to store information about the target image 

67 to-be-cleaned list; typically populated by hif_makeimlist. 

68 clean_list_pending: List of hif.tasks.makeimlist.cleantarget.CleanTarget 

69 objects representing what target images are to be cleaned; typically 

70 populated by hif_makeimlist, or edited by hif_editimlist. 

71 clean_masks: Dictionary to store the names of clean masks to be re-used 

72 for TARGET IQUV imaging. 

73 clean_thresholds: Dictionary to store thresholds to be re-used for 

74 TARGET IQUV imaging. 

75 contfile: Name of file with frequency ranges to use for continuum 

76 images; typically populated by hif_makeimlist. 

77 imaging_mode: Imaging mode string; may be used to switch between imaging 

78 parameter heuristics; currently only used for deciding what products 

79 to export. 

80 imaging_parameters: Dictionary containing computed values to use for 

81 certain parameters during imaging; typically populated by 

82 hifa_imageprecheck. 

83 linesfile: Name of file with line frequency ranges to exclude for 

84 continuum images; typically populated by hif_makeimlist. 

85 per_spw_cont_sensitivities_all_chan: Dictionary containing some imaging 

86 parameters as well as sensitivity information; typically populated 

87 by hifa_imageprecheck, and potentially updated by hif_makeimages or 

88 hif_tclean. 

89 processing_intents: Dictionary of processing intents for the current 

90 pipeline run. Currently, the following options are known: 

91 - {'INTERFEROMETRY_STANDARD_OBSERVING_MODE': 'Undefined'} 

92 - {'VLA_INTERFEROMETRY_STANDARD_OBSERVING_MODE': 'Undefined'} 

93 - {'INTERFEROMETRY_FULL_POL_CUBE_IMAGING': True} 

94 - {'INTERFEROMETRY_HETEROGENEOUS_IMAGING': True} 

95 - {'SINGLEDISH_STANDARD_OBSERVING_MODE': 'Undefined'} 

96 rmsimlist: The ImageLibrary object holding RMS uncertainty images of the 

97 science targets. 

98 sciimlist: The ImageLibrary object holding final images of science targets. 

99 selfcal_resources: List of files/tables required for the 

100 self-calibration restoration. 

101 selfcal_targets: List of targets for which self-calibration is performed. 

102 size_mitigation_parameters: Dictionary containing imaging product size 

103 mitigation parameters; typically populated by hif_checkproductsize, 

104 and used by hif_makeimlist. 

105 subimlist: The ImageLibrary object holding cut-out images of the science 

106 targets. 

107 synthesized_beams: Dictionary containing some imaging parameters as well 

108 computed synthesized beam information; typically populated/updated 

109 by hifa_imageprecheck, hif_checkproductsize, hif_makeimlist, 

110 hif_makeimages, and hif_tclean. 

111 """

283class Pipeline: 

284 """ 

285 Pipeline is the entry point for initialising the pipeline. It is 

286 responsible for the creation of new Context objects and for loading 

287 saved Contexts from disk. 

288 

289 Attributes: 

290 context: Context object containing the Pipeline state information. 

291 """ 

292 def __init__(self, context: str | None = None, loglevel: str = 'info', casa_version_check: bool = True, 

293 name: str | None = None, plotlevel: str = 'default', path_overrides: dict | None = None, 

294 processing_intents: dict = None): 

295 """ 

296 Initialise the pipeline, creating a new Context or loading a saved 

297 Context from disk. 

298 

299 Args: 

300 context: Filename of the pickled Context to load from disk. 

301 Specifying 'last' loads the last-saved Context, while passing 

302 None creates a new Context. 

303 loglevel: Pipeline log level. 

304 casa_version_check: enable (True) or bypass (False) the CASA version 

305 check. Default is True. 

306 name: If not "None", this overrides the name of the Pipeline Context 

307 if a new context needs to be created. 

308 plotlevel: Pipeline plots level. 

309 path_overrides: Optional dictionary containing context properties to 

310 be redefined when loading existing context (e.g. "name"). 

311 processing_intents: Dictionary of processing intents for the current 

312 pipeline run. 

313 """ 

314 # configure logging with the preferred log level 

315 logging.set_logging_level(level=loglevel) 

316 

317 # Prevent users from running the pipeline on old or incompatible 

318 # versions of CASA by comparing the CASA subversion revision against 

319 # our expected minimum and maximum 

320 if casa_version_check is True: 

321 if MIN_CASA_REVISION and environment.compare_casa_version('<', MIN_CASA_REVISION): 

322 msg = ('Minimum CASA revision for the pipeline is %s, ' 

323 'got CASA %s.' % (MIN_CASA_REVISION, environment.casa_version)) 

324 LOG.critical(msg) 

325 if MAX_CASA_REVISION and environment.compare_casa_version('>', MAX_CASA_REVISION): 

326 msg = ('Maximum CASA revision for the pipeline is %s, ' 

327 'got CASA %s.' % (MAX_CASA_REVISION, environment.casa_version)) 

328 LOG.critical(msg) 

329 

330 # if no previous context was specified, create a new context for the 

331 # given measurement set 

332 if context is None: 

333 self.context = Context(name=name) 

334 

335 # otherwise load the context from disk.. 

336 else: 

337 # .. by finding either last session, or.. 

338 if context == 'last': 

339 context = self._find_most_recent_session() 

340 

341 # .. the user-specified file 

342 with open(context, 'rb') as context_file: 

343 LOG.info('Reading context: %s', context) 

344 last_context = utils.pickle_load(context_file) 

345 self.context = last_context 

346 

347 event = ContextResumedEvent(context_name=last_context.name, output_dir=last_context.output_dir) 

348 eventbus.send_message(event) 

349 

350 # If requested, redefine context properties with given overrides. 

351 if path_overrides is not None: 

352 for k, v in path_overrides.items(): 

353 setattr(self.context, k, v) 

354 

355 if processing_intents is not None: 

356 self.context.processing_intents = processing_intents 

357 

358 self._link_casa_log(self.context) 

359 

360 # define the plot level as a global setting rather than on the 

361 # context, as we want it to be a session-wide setting and adjustable 

362 # mid-session for interactive use. 

363 import pipeline.infrastructure as infrastructure 

364 infrastructure.set_plot_level(plotlevel)

366 def _link_casa_log(self, context: Context) -> None: 

367 """ 

368 Create a hard-link to the current CASA log in the report directory, 

369 and add path to current CASA log to given Context. 

370 

371 Args: 

372 context: Pipeline Context to update with path to current CASA log. 

373 """ 

374 report_dir = context.report_dir 

375 

376 # create a hard-link to the current CASA log in the report directory 

377 src = casa_tools.log.logfile() 

378 dst = os.path.join(report_dir, os.path.basename(src)) 

379 if not os.path.exists(dst): 

380 try: 

381 os.link(src, dst) 

382 except OSError: 

383 LOG.error('Error creating hard link to CASA log') 

384 LOG.warning('Reverting to symbolic link to CASA log. This is unsupported!') 

385 try: 

386 os.symlink(src, dst) 

387 except OSError: 

388 LOG.error('Well, no CASA log for you') 

389 

390 # the web log creates links to each casa log. The name of each CASA 

391 # log is appended to the context. 

392 if 'casalogs' not in context.logs: 

393 # list as one casa log will be created per CASA session 

394 context.logs['casalogs'] = [] 

395 if src not in context.logs['casalogs']: 

396 context.logs['casalogs'].append(os.path.basename(dst))

398 @staticmethod 

399 def _find_most_recent_session(directory: str = './') -> str: 

400 """ 

401 Return filename for the most recently saved Pipeline Context in given directory. 

402 

403 Args: 

404 directory: Path where to search for context files. 

405 

406 Returns: 

407 Filename of most recently saved Pipeline Context file. 

408 

409 Raises: 

410 FileNotFoundError if no Pipeline context files are found in given directory. 

411 """ 

412 # list all the files in the directory.. 

413 files = [f for f in os.listdir(directory) if f.endswith('.context')] 

414 

415 if len(files) == 0: 

416 raise FileNotFoundError(f'No pipeline context exists in {os.path.abspath(directory)}') 

417 

418 # .. and from these matches, create a dict mapping files to their 

419 # modification timestamps, .. 

420 name_n_timestamp = dict([(f, os.stat(directory+f).st_mtime) 

421 for f in files]) 

422 

423 # .. then return the file with the most recent timestamp 

424 return max(name_n_timestamp, key=name_n_timestamp.get)
