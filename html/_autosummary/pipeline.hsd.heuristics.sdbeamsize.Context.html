pipeline.hsd.heuristics.sdbeamsize.Context

class Context(prec=None, rounding=None, Emin=None, Emax=None, capitals=None, clamp=None, flags=None, traps=None)

Bases: object

The context affects almost all operations and controls rounding, Over/Underflow, raising of exceptions and much more. A new context can be constructed as follows:

>>> c = Context(prec=28, Emin=-425000000, Emax=425000000,
...             rounding=ROUND_HALF_EVEN, capitals=1, clamp=1,
...             traps=[InvalidOperation, DivisionByZero, Overflow],
...             flags=[])
>>>

power(a, b, modulo=None)

Compute a**b. If ‘a’ is negative, then ‘b’ must be integral. The result will be inexact unless ‘a’ is integral and the result is finite and can be expressed exactly in ‘precision’ digits. In the Python version the result is always correctly rounded, in the C version the result is almost always correctly rounded.

If modulo is given, compute (a**b) % modulo. The following restrictions hold:

  • all three arguments must be integral

  • ‘b’ must be nonnegative

  • at least one of ‘a’ or ‘b’ must be nonzero

  • modulo must be nonzero and less than 10**prec in absolute value
