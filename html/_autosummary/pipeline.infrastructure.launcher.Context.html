pipeline.infrastructure.launcher.Context

Context holds all pipeline state, consisting of metadata describing the data set, objects describing the pipeline calibration state, the tree of Results objects summarising the results of each pipeline task, and a small number of internal pipeline variables and objects.

The aim of the Context class is to provide one central object to which all pipeline state is attached. Keeping all state in one object makes it easy to persist this one object, and thus all state, to disk as a Python pickle, allowing pipeline sessions to be interrupted and resumed.

ivar name:

Name of the context; also forms the root of the filename used for the pickled state.

ivar output_dir:

The working directory where pipeline output data should be saved.

ivar products_dir:

The directory where pipeline products should be exported to.

ivar report_dir:

The directory where pipeline HTML reports should be saved.

ivar logs:

Dictionary containing the filenames to use for standard output files (CASA commands log, AQUA report, pipeline script, restore script).

vartype logs:

dict[str, str | list]

ivar results:

The list of task result (proxy) objects containing summaries of each executed task in the pipeline run.

ivar task_counter:

Index of the last task to complete.

ivar subtask_counter:

Index of the last subtask to complete.

ivar observing_run:

The top-level ObservingRun object through which all other pipeline.domain objects can be accessed.

ivar project_performance_parameters:

ALMA OUS project performance information.

ivar project_structure:

ALMA project structure information.

ivar project_summary:

Project summary information.

ivar calimlist:

The ImageLibrary object holding final images of calibrators.

ivar callibrary:

The CalLibrary object holding the calibration state for registered measurement sets.

ivar clean_list_info:

Dictionary to store information about the target image to-be-cleaned list; typically populated by hif_makeimlist.

ivar clean_list_pending:

List of hif.tasks.makeimlist.cleantarget.CleanTarget objects representing what target images are to be cleaned; typically populated by hif_makeimlist, or edited by hif_editimlist.

ivar contfile:

Name of file with frequency ranges to use for continuum images; typically populated by hif_makeimlist.

vartype contfile:

str | None

ivar imaging_mode:

Imaging mode string; may be used to switch between imaging parameter heuristics; currently only used for deciding what products to export.

vartype imaging_mode:

str | None

ivar imaging_parameters:

Dictionary containing computed values to use for certain parameters during imaging; typically populated by hifa_imageprecheck.

ivar linesfile:

Name of file with line frequency ranges to exclude for continuum images; typically populated by hif_makeimlist.

vartype linesfile:

str | None

ivar per_spw_cont_sensitivities_all_chan:

Dictionary containing some imaging parameters as well as sensitivity information; typically populated by hifa_imageprecheck, and potentially updated by hif_makeimages or hif_tclean.

ivar rmsimlist:

The ImageLibrary object holding RMS uncertainty images of the science targets.

ivar sciimlist:

The ImageLibrary object holding final images of science targets.

ivar selfcal_resources:

List of files/tables required for the self-calibration restoration.

vartype selfcal_resources:

list[str]

ivar selfcal_targets:

List of targets for which self-calibration is performed.

ivar sensitivities:

TODO: introduced for CAS-10146, but appears unused.

ivar size_mitigation_parameters:

Dictionary containing imaging product size mitigation parameters; typically populated by hif_checkproductsize, and used by hif_makeimlist.

ivar subimlist:

The ImageLibrary object holding cut-out images of the science targets.

ivar synthesized_beams:

Dictionary containing some imaging parameters as well computed synthesized beam information; typically populated/updated by hifa_imageprecheck, hif_checkproductsize, hif_makeimlist, hif_makeimages, and hif_tclean.

Module attributes

products_dir

Return path to the products directory.

stage

Return task and sub-task stage number.

vla_skip_mfs_and_cube_imaging

Return the stage skipping condition for the VLA specmode=mfs/cube imaging workflow.
