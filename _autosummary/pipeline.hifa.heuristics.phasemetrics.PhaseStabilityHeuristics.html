Do an averaging/smoopthing on the phase data The default for ALMA for phase statistics should be 10s. For default 6s integration times this will average 2 values. If input phases from the gain table have integration(diffTime) > 10s then no averaging is made.

static std_overlapping_avg(phase: ndarray, diffTime: float, over: float = 120.0) → float[source]ÔÉÅ

Calculate STD over a set time and return the average of all overlapping values of the standard deviation - overlapping estimator. This acts upon a phase-time stream of data. Will consider only finite values in the phase-time stream (i.e. phase array). The phase array must be unwrapped, i.e. continous, no breaks or 2PI ambiguities

This is the standard deviation, so it takes out the mean value. RMS with mean or fit removed provides the same value for zero-centered phases.

Parameters:

  • phase (array) ‚Äì any unwrapped input phase

  • diffTime (float) ‚Äì the time between each data integration

  • over (float) ‚Äì time in seconds to calculate the SD over

Returns:

average standard deviation for the dataset calcualted over the input timescale

Return type:

float
