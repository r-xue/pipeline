aggregate_fully_flagged_antenna_notifications

Aggregate the list of notifications about fully flagged antennas for the subsequent QA scoring. :Parameters: * fully_flagged_antennas -- dict with tuples(field, intent, spwid) as keys and sets of antenna ids as values. * all_spwids -- set of all spw ids considered in this flagging task, used to replace the tuple of flagged spws by a string "all spws" when this is the case.

format_fully_flagged_antenna_notification

Format the notification message for fully flagged antennas. :Parameters: * vis -- the name of the ms. * notification -- the tuple containing the information about fully flagged antennas.

identify_fully_flagged_antennas_from_flagcmds

Identify the set of antennas that were fully flagged in at least one spw considered in the given flagging task, for any of the fields or intents, by analyzing the flagging commands. :Parameters: * ms -- measurement set. * flags -- list of flagging commands.

identify_fully_flagged_antennas_from_flagview

Identify the set of antennas that were fully flagged in at least one spw considered in the given flagging task, for any of the fields or intents, by analyzing the flagging view. :Parameters: * ms -- measurement set. * results -- a FlaggableViewResults object containing one or more flagging views, which are two-dimensional arrays with 0th axis enumerating the scans and 1st axis enumerating the antennas. * scan_to_field_names -- a dictionary providing the translation between scans and fields. the scans (0th axis) may be enumerated not only by scan index, but by any other parameter that uniquely determines the scan (such as the timestamp), which serves as the dict key, while its value contains the field(s) corresponding to each scan. * intents_of_interest -- if provided, lists the intents to be considered in this analysis.

mark_antennas_for_refant_update

Modify result to set antennas to be demoted/removed if/when result gets accepted into the pipeline context.
