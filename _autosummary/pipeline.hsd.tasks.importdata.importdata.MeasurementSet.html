get_diffgain_mode() → str | None[source]ÔÉÅ

Determine if the intents and SpW setup in this measurement set are consistent with an observing mode that uses a differential gain calibrator: BandToBand (B2B) or BandwidthSwitching (BWSW).

Returns:

‚ÄòB2B‚Äô if the ratio of frequencies between on-source and reference spws is above 1.661; ‚ÄòBWSW‚Äô if the ratio of bandwidths between reference and on-source is above 1.5; None if there are no DIFFGAIN* intents in this MS.

Raises:

  • ValueError if the DIFFGAIN* intents are present in the MS, but either ‚Äì

  • a. the MS is missing diffgain reference or on-source SpWs ‚Äì

  • b. the SpW setup does not match either BandToBand or BandwidthSwitching. ‚Äì

Set data type and column.

Set data type and column to MS domain object and record the available data types per (source,spw) tuple. If source or spw are unset, they will be expanded to all available values.

Parameters:

  • dtype ‚Äì data type to set

  • column ‚Äì name of column in MS associated with the data type

  • source ‚Äì source name selection string (comma separated names). If unset, all sources will be used.

  • spw ‚Äì real spectral window selection string (string of comma separated IDs). If unset, all real spw IDs will be used.

  • overwrite ‚Äì if True existing data colum is overwritten by the new column. If False and if type is already associated with other column, the function raises ValueError.

Raises:

ValueError ‚Äì An error raised when the column does not exist or the type is already associated with a column or the column is already assigned to a type and would not be overwritten.
