The timetracker module contains classes that track execution duration based on the reception of lifecycle events.

Classes

ContextCreatedEvent

Emitted when a pipeline Context is created.

ContextLifecycleEvent

Base class for events related to the lifecycle of the pipeline context.

ContextResumedEvent

Emitted when a pipeline Context is resumed.

ContextTimeTracker

ContextTimeTracker listens for events related to the creation/resumption of a pipeline Context.

ResultAcceptErrorEvent

ResultAcceptedEvent

Emitted when a Result has been successfully incorporated into the Context.

ResultAcceptingEvent

Emitted when a Result is accepted into the Context.

ResultLifecycleEvent

Base class for events related to the acceptance of a pipeline Result.

TaskAbnormalExitEvent

Emitted when a pipeline Task terminates due to an unexpected error.

TaskCompleteEvent

Emitted when a pipeline Task completes.

TaskLifecycleEvent

Base class for events related to the pipeline task lifecycle.

TaskStartedEvent

Emitted when a pipeline Task is started.

TaskTimeTracker

TaskTimeTracker listens for pipeline lifecycle events, recording the start and end times on event reception so that the duration of the corresponding pipeline execution phase can be calculated.

WebLogStageLifecycleEvent
